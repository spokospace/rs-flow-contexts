From discovery to first commit — automated

Rocksoft-IT · 2026

The problem

  • Every new project starts with 6–8 identical manual steps
  • Discovery notes, repo creation, context setup, issue backlog — done by hand every time
  • Context lives in someone's head or a random Notion page
  • Onboarding a second dev takes longer than it should

The full flow

n8n
 └─ generates discovery notes (.md)
     └─ rs-flow-contexts (GitHub repo)
         └─ repo-request .md added
             └─ create-repos.yml (GitHub Action)
                 ├─ gh repo create
                 ├─ git submodule add rs-skills → .claude/skills/
                 ├─ copies context/ into new repo
                 └─ pushes bootstrap.yml workflow
                      └─ [trigger] claude-code-action
                           ├─ reads discovery-notes.md
                           ├─ creates GitHub Issues (FR-001…)
                           └─ creates feature branches

rs-flow-contexts

The control plane for all project repos

  • One .md file = one repo request
  • Frontmatter drives everything: name, visibility, topics, context_path
  • Idempotent — safe to re-run, existing repos are skipped
  • n8n writes context → dev adds a .md → Action runs

repo: meniu-qr
description: QR table-ordering web app for cafés
visibility: private
context_path: 2026-06-05-meniu-qr-self-funded-startup/context
topics: [rs-flow, discovery, greenfield, web-app]

rs-skills

Claude Code as a dev tool — local workflow

  • Set of slash commands for Claude Code CLI
  • Installed as a git submodule at .claude/skills/ in every project repo
  • Available from the first clone — no manual setup

/rs-init — scaffold context/
/rs-shape — structure the idea
/rs-prd — write the PRD
/rs-roadmap — sequence delivery
/rs-plan — plan a feature
/rs-implement — implement it
/rs-impl_review — review code
/rs-archive — close the change

Works with a claude.ai Pro/Max subscription — no API key needed locally

claude-code-action

Claude Code running inside GitHub Actions

  • anthropics/claude-code-action@v1 — 7 900+ stars, production-ready
  • Reads the repo, creates issues, writes code, opens PRs
  • Trigger options: workflow_dispatch, label, @claude in a comment
  • Workflow file pushed automatically during provisioning
  • Org-level secret ANTHROPIC_API_KEY → zero per-repo config

- uses: anthropics/claude-code-action@v1
  with:
    anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
    prompt: |
      Read context/foundation/discovery-notes.md.
      For every FR-XXX create a GitHub Issue + branch feature/FR-XXX.

Optimizing Routines

One Routine = one run counted.
Batch /rs-roadmap + /rs-prd into a single Routine.
Schedule during off-hours → 25 runs/day covers ~3 projects/hour.

Recommended split

Local dev → subscription (free)
Automation → API key (~$2/month for 40 runs)
Routines → use for monitoring / daily digest tasks

Azure VM limitation sources: Claude Code Headless Mode — OAuth requires browser, API key required for unattended runs · GitHub #44267: OAuth token expires in CI · GitHub #33897: OAuth hangs without browser
